1.项目简介
     本项目是一个基于 STM32F103 的裸机手写字符识别系统。系统通过电容触摸屏高频采集用户的手写轨迹，可以实时显示笔迹轨迹的效果。并将轨迹数据流送入特征识别引擎，最终输出匹配的英文字母或数字串。
    本项目的底层硬件驱动（如 LCD 显示、GT9147 触摸芯片控制）以及 核心识别算法库均来源于网上开源生态。我的核心工作在于系统的二次重构、模块整合、电容屏特性的逻辑适配，以及整体数据流转与动态内存调度的调试优化。

2.软件架构
user
包含 main.c 主循环，负责状态机的流转、按键模式切换、轨迹点的记录过滤以及 UI 绘图引擎的调度。

CR ( Character Recognition)
字符识别模块。封装了算法库的接口，负责接收采集到的坐标点阵，进行特征匹配并返回识别字符串。

hardware
touch：基于IIC协议的GT9147电容触摸屏驱动，向上层提供纯净的 x, y 坐标数据。
lcd：基于FSMC总线的TFT-LCD驱动，实现了内存映射级别的高速屏幕刷新。
key：实体按键驱动，用于切换识别模式（数字/大写/小写/混合）。

malloc
重要的支撑模块。实现了一套分块式的堆内存池（Memory Pool），专供识别算法在运行时动态申请和释放内存，彻底避免裸机环境下的内存碎片化和堆栈溢出。

system
包含基于 SysTick 的精准硬件延时（delay）、串口调试接口（usart）以及 Cortex-M3 内核的底层位带操作（sys）。

3.运行流程 
数据采集与过滤 
     main.c 的主循环以极高的频率调用touch.c的扫描接口。当用户手指接触屏幕时，GT9147芯片通过IIC 将物理触控点转化为数字坐标。在应用层中，我加入了防抖与去重逻辑：只有当当前坐标与上一个坐标发生有效偏移时，才将其存入READ_BUF轨迹数组中，避免原地按压导致的数据冗余。

实时轨迹渲染 
    在采集坐标的同时，main.c 会调用绘图函数（lcd_draw_bline）。该函数通过连续绘制重叠的实心（gui_fill_circle），将相邻的两个离散坐标点连接起来，并写入LCD的显存中，从而在屏幕上呈现出平滑且带有粗细的手写视觉效果。

动态内存调度 
    当用户抬手并经过设定的超时时间后，系统触发识别流程。此时，CR模块会被唤醒。为了节省MCU 的静态RAM，CR 模块会向malloc内存池临时申请一块连续的大内存，用于装载特征匹配模型。

模式匹配与输出 
    CR算法引擎对READ_BUF中的轨迹点阵进行降噪、归一化计算，并结合用户当前选择的识别模式，输出相似度最高的ASCII字符串结果。
最终，结果被传递给LCD驱动显示在屏幕的Result区域，同时malloc模块立即回收刚才申请的内存，系统清屏并等待下一次输入。
